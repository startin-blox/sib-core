EDC Federated Catalog Demo

Displaying consolidated assets from one of our EDC connectors

Federated Demo Configuration

This demo fetches and consolidates catalogs from multiple EDC connectors:

  • Consumer Connector: consumer.connector-dev.startinblox.com
  • Provider Connectors:
    • SB Provider: provider.connector-dev.startinblox.com/protocol
  • Protocol: Dataspace Protocol HTTP (EDC v3 Management API)

Key Features:

  • ✅ Parallel catalog fetching from multiple connectors
  • ✅ Dataset deduplication across providers
  • ✅ Provider identification with color-coded badges
  • ✅ Per-provider statistics and health monitoring
  • ✅ Advanced filtering by provider and search terms
  • ✅ Provider-aware contract negotiation
  • ✅ Graceful error handling when some connectors fail

This is equivalent to running multiple parallel requests:

# Request 1 - Dreamwall
curl -X POST "https://consumer.connector-dev.startinblox.com/management/v3/catalog/request" \
    -H 'Content-Type: application/json' \
    -d '{"@context":{"@vocab":"https://w3id.org/edc/v0.0.1/ns/"},"counterPartyAddress":"https://provider.connector-dev.startinblox.com/protocol","protocol":"dataspace-protocol-http"}'
