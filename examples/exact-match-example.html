üìã Overview

This example demonstrates how to use the enhanced store.generateShapes method with exact matching capabilities for specific fields like tc:country and dcat:format.

Key Feature: The exactMatchMapping parameter allows you to specify which properties should use exact matching (sh:hasValue) instead of pattern matching (sh:pattern).

üîß Usage Example

1. Basic Query with Pattern Matching (Default)

// Standard query - uses sh:pattern with ".*" suffix const options: IndexQueryOptions = { dataSrcIndex: "http://localhost:5173/data/ai/3DObjects-index.jsonld", dataRdfType: "tc:3DObject", filterValues: { "tc:title": { value: "castle" } } }; const results = await store.queryIndex(options); // Generates: sh:pattern "castle.*"

2. Query with Exact Matching

// Query with exact matching for specific fields const options: IndexQueryOptions = { dataSrcIndex: "http://localhost:5173/data/ai/3DObjects-index.jsonld", dataRdfType: "tc:3DObject", filterValues: { "tc:title": { value: "castle" }, "tc:country": { value: "France" }, "dcat:format": { value: "3D" } }, exactMatchMapping: { "tc:country": "France", // Uses sh:hasValue instead of sh:pattern "dcat:format": "3D" // Uses sh:hasValue instead of sh:pattern } }; const results = await store.queryIndex(options); // Generates: // - tc:title: sh:pattern "castle.*" // - tc:country: sh:hasValue "France" // - dcat:format: sh:hasValue "3D"

3. Conjunction Query with Exact Matching

// Conjunction query with exact matching const conjunctionOptions: ConjunctionQueryOptions = { dataSrcIndex: "http://localhost:5173/data/ai/3DObjects-index.jsonld", dataRdfType: "tc:3DObject", filterValues: { "tc:title": { value: "medieval" }, "tc:country": { value: "Germany" }, "dcat:format": { value: "gltf" } }, useConjunction: true, exactMatchMapping: { "tc:country": "Germany", "dcat:format": "gltf" } }; const results = await store.queryIndexConjunction(conjunctionOptions);

üéØ Use Cases

When to Use Exact Matching

  • Enumerated values: Countries, formats, licenses, categories
  • Precise identifiers: Codes, IDs, specific names
  • Controlled vocabularies: Standardized terms

When to Use Pattern Matching

  • Text search: Titles, descriptions, content
  • Partial matches: User input, search terms
  • Fuzzy search: Approximate matching
Example: For 3D models, you might want exact matching for dcat:format ("gltf", "fbx", "obj") but pattern matching for tc:title to allow partial text searches.

üöÄ Benefits

  • Flexibility: Choose the right matching strategy for each field
  • Performance: Exact matches can be more efficient than pattern matching
  • Accuracy: Avoid false positives from partial matches on enumerated values
  • Backward Compatibility: Existing code continues to work without changes
